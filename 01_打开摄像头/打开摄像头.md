# 嵌入式 Linux 视频采集：V4L2 入门开发指南

## 第一章：理解 V4L2 架构与设备节点

在 Linux 操作系统中，万物皆文件。Video4Linux2 (V4L2) 是 Linux 处理视频设备（如采集卡、USB 摄像头、电台等）的核心内核框架。它向上层应用程序提供了一套统一的 API，屏蔽了底层复杂多样的硬件差异。

**1. 核心概念：设备节点**
当系统成功加载视频设备的底层驱动后，会在 `/dev` 目录下暴露出相应的字符设备节点，通常命名为 `/dev/video0` 到 `/dev/video63`。对于基础的视频采集应用，主要交互对象就是代表视频流捕获的 `/dev/videoX` 节点。

**2. 核心系统调用**
应用程序不需要直接编写复杂的硬件驱动，而是通过标准的 Linux 文件 I/O 操作与内核层进行通信。最基础的操作包括：
*   `open()`：打开视频设备节点。
*   `ioctl()`：发送控制命令，与设备进行“握手”配置。
*   `mmap()`：将内核空间分配的缓冲区映射到用户空间，实现**零拷贝（Zero-copy）**的数据传输。
*   `close()`：关闭设备并释放资源。

## 第二章：使用 v4l-utils 工具链探索硬件

在直接编写 C 语言代码之前，**工业界的标准做法是先使用 `v4l-utils` 提供的命令行工具，从外部扫描和验证硬件能力**，避免在代码中盲目试错。

### 2.1 扫描并发现设备
要确定系统是否识别到了摄像头以及它的设备节点分配情况，可以使用以下命令：
```bash
v4l2-ctl --list-devices
```
该命令会列出所有注册的 V4L2 设备及其挂载的物理总线（如 USB）。

### 2.2 查询设备格式与分辨率
不同的摄像头支持的输出格式（如无损的 YUYV 或压缩的 MJPEG）以及对应的分辨率和帧率各有不同。利用如下命令可以彻底摸清摄像头的“底细”：
```bash
v4l2-ctl --list-formats-ext
```
此命令将打印出每种像素格式下，硬件原生支持的分辨率大小和帧间隔（即帧率）。

### 2.3 动态调节摄像头参数
V4L2 暴露了丰富的控制（Control）接口。你可以查询并实时调整摄像头的亮度、对比度、曝光等参数：
```bash
v4l2-ctl --list-ctrls         # 列出所有支持的控制项及其取值范围
v4l2-ctl -c brightness=128    # 将亮度设置为 128
```

## 第三章：编写首个 C/C++ 采集程序

本章将详述如何通过 C/C++ 调用 V4L2 API 抓取一帧原始图像。其核心思想是**“缓冲区轮转”（Buffer Lifecycle）**机制。

### 3.1 打开设备
首先，使用标准的文件打开操作获取文件描述符（File Descriptor）：
```c
int fd = open("/dev/video0", O_RDWR);
```
如果成功，`fd` 将作为后续所有 `ioctl` 调用的句柄。

### 3.2 查询设备能力 (Query Capabilities)
打开设备后，必须确认该设备是否支持**视频捕获 (Video Capture)** 和 **流式 I/O (Streaming I/O)**：
```c
struct v4l2_capability cap;
ioctl(fd, VIDIOC_QUERYCAP, &cap);
```
需要检查 `cap.capabilities` 标志位中是否包含 `V4L2_CAP_VIDEO_CAPTURE` 和 `V4L2_CAP_STREAMING`。

### 3.3 协商并设置格式 (Set Format)
明确告诉底层驱动你期望的画面尺寸和像素格式（例如宽 640、高 480，格式为 YUYV）：
```c
struct v4l2_format fmt;
fmt.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
fmt.fmt.pix.width = 640;
fmt.fmt.pix.height = 480;
fmt.fmt.pix.pixelformat = V4L2_PIX_FMT_YUYV;
ioctl(fd, VIDIOC_S_FMT, &fmt);
```
驱动可能会根据硬件的实际能力对你请求的分辨率进行向下对齐或调整，因此 `ioctl` 执行后 `fmt` 结构体中会返回实际生效的参数。

### 3.4 申请与映射缓冲区 (Request & Map Buffers)
为了极大地提高性能，现代 V4L2 程序通常**使用 `mmap` 内存映射模式，让用户空间直接访问内核内存**。
1.  **申请内存**：向内核请求分配指定数量（如 4 个）的缓冲区。
    ```c
    struct v4l2_requestbuffers reqbuf;
    reqbuf.count = 4;
    reqbuf.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
    reqbuf.memory = V4L2_MEMORY_MMAP;
    ioctl(fd, VIDIOC_REQBUFS, &reqbuf);
    ```
2.  **查询并映射**：循环遍历每一个缓冲区，查询其物理偏移量并映射到用户空间指针。
    ```c
    struct v4l2_buffer buf;
    buf.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
    buf.memory = V4L2_MEMORY_MMAP;
    buf.index = i;
    ioctl(fd, VIDIOC_QUERYBUF, &buf);
    void *start = mmap(NULL, buf.length, PROT_READ | PROT_WRITE, MAP_SHARED, fd, buf.m.offset);
    ```

### 3.5 图像采集循环 (Queue & Dequeue)
一切就绪后，启动硬件数据流并开始抓取：
1.  **初始入队 (Queue)**：将刚才映射好的所有空缓冲区放入驱动的处理队列中。
    ```c
    ioctl(fd, VIDIOC_QBUF, &buf);
    ```
2.  **启动数据流**：命令硬件开始工作，传感器信号开始填充缓冲队列。
    ```c
    int type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
    ioctl(fd, VIDIOC_STREAMON, &type);
    ```
3.  **等待并出队 (Wait & Dequeue)**：使用 `select()` 或 `poll()` 阻塞等待设备就绪。一旦硬件填满一个缓冲区，就将其“拔出”：
    ```c
    ioctl(fd, VIDIOC_DQBUF, &buf);
    ```
4.  **处理数据与重新入队**：此时图像数据已躺在对应的 `start` 指针中，你可以直接将其保存为文件：
    ```c
    fwrite(buffers[buf.index].start, 1, buf.bytesused, file);
    ```
    **关键一步：** 处理完数据后，必须立刻用 `VIDIOC_QBUF` 将该缓冲区还给内核，以供下一次循环填充，否则会导致设备无缓冲可用而卡死。

### 3.6 停止与资源释放 (Stream Off & Cleanup)
当程序结束时，需妥善释放系统资源以避免内存泄漏或设备锁死：
```c
ioctl(fd, VIDIOC_STREAMOFF, &type); // 停止采集
munmap(start, length);              // 解除内存映射
close(fd);                          // 关闭设备节点
```
